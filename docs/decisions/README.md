# Architecture Decision Records (ADRs)

Elke significante technische beslissing wordt vastgelegd als ADR.

## Formaat

```
ADR-[nummer]-[korte-naam].md
```

Voorbeeld: `ADR-001-stack-beslissingen.md`

## Structuur van een ADR

```markdown
# ADR-[nummer]: [Titel]

**Status:** proposed | accepted | superseded | deprecated
**Datum:** [datum]
**Superseded door:** [link naar nieuw ADR, indien van toepassing]

## Context
Waarom moeten we een beslissing nemen?

## Beslissing
Wat hebben we besloten?

## Consequenties
Wat volgt uit deze beslissing — positief en negatief?
```
